# v1.0.0 Release Notes

`monitor` v1.0.0 is the first stable release of the lightweight Go service monitor middleware.

## Highlights

- Provides a tiny `net/http` middleware with an embedded HTML status page and JSON snapshot endpoint.
- Tracks real-time process, Go runtime, host, disk, and HTTP request metrics for the current service instance.
- Keeps the runtime model intentionally lightweight: no server-side history, no external frontend assets, no storage, and no background network services.
- Best suited for single-node monolithic Go services that need quick local visibility into basic service health.

## Added

- Embedded monitor page at `Config.Path`, defaulting to `/monitor`.
- JSON snapshot responses through `Accept: application/json`, plus `APIOnly` mode for JSON-only endpoints.
- Business request counters for total requests, in-flight requests, status code classes, and latency.
- Process metrics including CPU, RSS, PID, thread count, and file handle count when available.
- Go runtime metrics including goroutines, goroutine peak, heap allocation, heap system memory, heap object count, GC count, GC pause timing, allocation count, free count, next GC target, and uptime.
- Host metrics including CPU, memory, load, disk summary, and optional multi-disk details through `DiskPaths`.
- Short in-browser trend charts powered by native Canvas.
- English and Simplified Chinese UI.
- Configurable page title, description, footer, default language, default theme, sample window, refresh interval, and background style.
- Light and dark themes, with dark as the default theme.
- Optional `grid` background style in addition to the default `solid` background.
- `IgnoreRequest` hook for excluding health checks and other non-business traffic from request counts.
- `NewMonitor` for reading the current snapshot through `Current()` and stopping the collector with `Stop()`.
- Documentation for using `monitor` with Fiber through Fiber's official `adaptor.HTTPMiddleware`.
- Chinese documentation under `docs/zh`.

## Operational Notes

- Metrics are collected in memory and reset when the process restarts.
- The monitor endpoint itself is excluded from business request counters.
- The monitor path accepts only `GET` and `HEAD`.
- Responses set `Cache-Control: no-store`, `Referrer-Policy: no-referrer`, and `X-Content-Type-Options: nosniff`.
- `Monitor` is safe for concurrent use after construction.

## Not Included

- Long-term metric storage.
- Alerting.
- Distributed tracing.
- Multi-instance aggregation.
- Application-specific business metrics.
- Prometheus or Grafana replacement features.

## Upgrade Notes

- This is the first stable release. Users on pre-v1 revisions should review the `Config` defaults and supported values before upgrading.
- `DefaultTheme` supports `light` and `dark`; empty or unsupported values fall back to `dark`.
- `Background` supports `solid` and `grid`; empty or unsupported values fall back to `solid`.
