# 安全政策

## 支持版本

安全修复会提供给最新发布版本和当前 main 开发分支。

## 报告漏洞

请不要在公开 issue 中报告安全漏洞。

如果本仓库启用了 GitHub 私密漏洞报告，请优先使用该功能。如果不可用，请创建一个最小化的公开 issue，请求私密联系方式，不要包含利用细节、日志、token 或敏感环境信息。

报告时请尽量包含：

- 受影响的版本或提交
- 简洁的问题描述
- 复现步骤或 proof of concept
- 预期影响
- 已知缓解方式

## 响应

这是一个小型开源项目，没有正式的安全响应 SLA。维护者会尽力确认有效报告、调查问题，并在适当时发布修复或缓解方案。

## 范围

`monitor` 是一个轻量 `net/http` 中间件。以下类型的安全报告最有帮助：

- 敏感运行时数据暴露
- 监控端点中的不安全 HTTP 行为
- 可能影响被包装服务的请求处理 bug
- 对本包有实际影响的依赖漏洞
