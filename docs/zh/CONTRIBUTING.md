# 贡献指南

感谢你帮助改进 `monitor`。

这个项目刻意保持很小：一个 `net/http` 中间件、一个状态页、一个 JSON 快照。请让改动聚焦在这个范围内。

## 开发

要求：

- Go 1.24 或更新版本

提交 pull request 前，请运行：

```sh
test -z "$(gofmt -l .)"
go test ./...
go vet ./...
```

如果改动涉及并发敏感逻辑，也请运行：

```sh
go test -race ./...
```

## 设计准则

- 保持公开 API 小而易用。
- 优先使用 Go 标准库；只有依赖能带来明确价值时才引入。
- 不要加入前端构建工具或外部浏览器资源。
- 不要把监控端点请求计入业务流量。
- 保持请求路径上的额外开销很低。
- 为可复用的导出类型记录并发行为。
- 可选指标采集失败时返回有用的零值。

## Pull Requests

请包含：

- 简短的变更说明
- 行为变更对应的测试
- 当公开行为或配置发生变化时，同步更新文档

请避免把无关重构和功能 / bug 修复混在同一个 PR 中。
